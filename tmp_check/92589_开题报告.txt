选题背景与意义 /     随着5G的普及和6G愿景的提出，万物互联导致全球数据流量呈指数级增长，其中图像和视频数据占比极大。传统的通信系统设计遵循香农分离定理（Shannon’s Separation Theorem），将信源编码（压缩）与信道编码（纠错）作为两个独立的模块进行优化。虽然这种架构在长码块长度下理论上是最优的，但在低信噪比、带宽受限或对延迟敏感的无线信道环境下，其性能面临严重瓶颈。特别是著名的“悬崖效应”（Cliff Effect），即当信道条件低于设计阈值时，传统数字系统的性能会急剧下降导致通信中断。 /     语义通信作为6G的核心技术之一，提出了一种全新的范式：通信系统不应仅仅关注比特的无误传输，而应关注信息的“意义”或“语义”的成功传递。通过深度学习技术，让通信系统直接提取和传输图像的深层语义特征，不仅能大幅压缩数据量，还能在恶劣信道下保持较好的鲁棒性。 /     本课题旨在设计并实现一种基于深度语义通信的图像传输系统，通过端到端的联合信源信道编码（JSCC）架构，提升图像在复杂无线信道下的传输质量与鲁棒性，并开发可视化演示系统。这对于推动下一代智能通信系统的发展具有重要的理论意义和工程应用价值。 / 国内外研究现状 /     传统通信方案：目前主流的图像传输方案通常采用JPEG、JPEG2000或BPG等标准进行压缩，再结合LDPC、Turbo码等信道编码及QAM调制进行传输。这种“分离设计”方案在信道状况良好时表现优异，但在低信噪比或快衰落信道下，由于信源压缩对误码极度敏感，往往需要大量的纠错冗余，导致传输效率低下且容易崩溃。 /     国外研究现状：近年来，基于深度学习的端到端通信成为热点。Bourtsoulatze等人2019年提出了DeepJSCC框架，利用卷积神经网络（CNN）将图像像素直接映射为信道输入符号，证明了在低SNR下其性能显著优于“JPEG+LDPC”方案，且具有优雅的性能降级特性（Graceful Degradation），避免了悬崖效应。Xie等人2021年提出了DeepSC框架，基于Transformer架构实现了文本的语义传输，并提出了语义相似度指标。 /     国内研究现状：国内在语义通信领域发展迅速。清华大学团队（陆建华院士、陶晓明教授等） 是国内最早系统性研究语义通信的课题组之一。他们重点关注面向任务的语义通信（Task-oriented Semantic Communication），即通信不仅仅是为了恢复数据，而是为了更好地完成下游任务（如分类、检测）。其团队提出了基于深度神经网络的语义通信系统模型，验证了在文本和图像传输中，利用语义特征进行传输可以显著降低所需带宽。 /     北京邮电大学团队（张平院士、许文俊教授等）提出了“智简语义通信”的概念，并成功搭建了国际上首个面向6G的语义通信原型演示系统。他们的研究不仅局限于仿真，更注重系统在实际硬件上的落地。在图像和视频传输方面，该团队针对语义特征在无线信道中的畸变问题，设计了多种基于知识图谱（Knowledge Graph）辅助的语义纠错机制，有效解决了语义“歧义性”问题。 /     国内目前的研究现状更侧重于“模型优化与系统落地”，即如何解决模型在变信道下的泛化性差、计算开销大以及如何从单一模态向多模态（视听融合）演进的问题。 /     发展趋势：当前研究正从单一模态（文本）向多模态（图像、视频）扩展，从纯理论仿真向原型系统与可视化演示迈进。未来的趋势是结合注意力机制进一步提升语义提取效率，并结合可微信道模型实现对实际无线环境的自适应。 | 选题背景与意义 / 2.1研究目标 / 本课题旨在基于深度学习框架DeepSC，设计并实现一套面向图像任务的端到端语义编码传输系统。具体目标包括： / 构建一个鲁棒的图像语义通信模型，在低信噪比（如-5dB至5dB）和瑞利衰落信道下，其图像重建质量（PSNR/SSIM）显著优于传统的“JPEG/BPG + 调制信道”方案。 / 实现端到端的可微信道建模，使系统具备对抗加性高斯白噪声和衰落的能力。 / 开发一套完整的图形用户界面（GUI）软件，支持实时图像传输演示、信噪比（SNR）调节及性能指标自动计算。 / 2.2主要研究内容 / 语义编解码器网络设计： 基于深度语义通信框架，构建包含卷积层与注意力机制的语义编码器，将高维图像提取为低维的模拟语义特征符号；设计对应的语义解码器以重构图像。重点研究不同压缩率（带宽压缩比）对重建质量的影响。 / 信道建模与鲁棒性增强： 3.1研究方法 / 模型构建与训练： 采用 PyTorch 深度学习框架搭建端到端网络。使用 Adam 优化器进行模型参数更新。损失函数（Loss Function）将采用 MSE（均方误差） 与 SSIM（结构相似性） 的混合损失函数，以兼顾像素级准确度和视觉感知质量。 / 仿真实验法： 选用 CIFAR-10（用于快速验证）和 Kodak（用于高质量评估）图像数据集。采用蒙特卡洛仿真方法，在训练中引入随机信道噪声，增强模型的泛化能力。 / 模块化开发： 将训练好的模型封装为ONNX或TorchScript推理接口，集成到GUI软件中。 / 分析验证：通过曲线分析、可视化对比等方式验证模型在不同信道条件下的可靠性与工程可行性。 / 3.2实施步骤 / 数据集准备与预处理阶段：配置Python、PyTorch及CUDA环境，选用 CIFAR-10 数据集作为核心训练集，以快速验证网络结构有效性；选用 Kodak 无损图像数据集作为最终测试集，用于评估系统在真实高分图像下的泛化性能。对训练数据进行归一化、随机裁剪和水平翻转处理，以增强模型的鲁棒性，防止过拟合。 / 模型设计与仿真：复现DeepJSCC基础模型，加入注意力模块改进编码器，嵌入AWGN/Rayleigh信道层进行联合训练。 / 基线系统实现：编写脚本模拟传统“JPEG+信道编码”的传输过程，记录该基线系统在不同信噪比下的误码率及图像解码成功率，作为对比基准。 / 软件系统集成：使用Streamlit或PyQt编写前端界面，后端调用模型推理接口，实现交互式功能。前端功能实现图像加载、信道参数（SNR值）滑块调节、传输模拟按钮；设计双屏显示区域，左侧显示原图，右侧实时显示经过DeepSC重构的图像和传统方案重构的图像。 / 测试与论文撰写：在AWGN和瑞利衰落信道下，分别测试本系统与基线系统的性能；收集实验数据，绘制PSNR/SNR性能曲线；分析语义特征传输相较比特流在被窃听环境下的解码难度与潜在隐私保护能力；分析语义通信减少重传、冗余编码带来的能耗节省；分析结果，撰写毕业论文。 / 3.3拟解决的主要问题及措施 / 在研究与实现过程中，预计会遇到以下关键技术难点，拟采取如下措施解决： / 物理信道的不可微性导致的端到端训练困难：实际无线信道（如衰落、硬件损伤）通常包含复杂的随机过程，直接将物理信道模型嵌入神经网络会导致梯度无法反向传播，无法进行端到端优化。 / 解决措施：采用 “生成式信道建模” 或 “重参数化技巧”。在训练阶段，不直接调用物理设备，而是使用数学解析模型来模拟信道传输过程，确保整个链路在数学上是可导的。同时，在推理/演示阶段，保持模型参数不变，验证其在模拟环境下的有效性。 / 深度语义模型在不同信噪比下的泛化能力差：通常针对特定SNR训练的模型，在信道条件改变时性能会大幅下降（即模型对SNR失配敏感）。若为每个SNR点单独训练模型，存储开销大且不实用。 / 解决措施：引入注意力机制辅助的信噪比感知网络。将当前的SNR值作为一个外部特征向量输入到编码器和解码器中，利用注意力模块动态调整网络权重。这使得单一模型能够感知信道状态，自动调整编码策略。 / 图像视觉质量评价与语义一致性的平衡：传统的MSE损失函数倾向于生成模糊平滑的图像，虽然信噪比数值尚可，但丢失了纹理细节（即视觉语义较差），导致“指标高但观感差”。解决措施：在损失函数中引入 SSIM（结构相似性）损失和感知损失（Perceptual Loss, 基于VGG网络提取特征），强制网络关注图像的高层语义特征而非仅仅是像素值差异。在GUI设计中，除了展示PSNR数值外，增加SSIM指标展示，并保留人工主观评估的窗口，确保系统优化的方向符合人类视觉感知。 / 复杂神经网络的计算开销与实时性矛盾：基于Transformer或深层CNN的模型计算量巨大，任务书要求在普通PC上实现低延迟推理，这对计算效率提出了挑战。 / 解决措施：模型轻量化：采用深度可分离卷积替代标准卷积，减少参数量。推理加速：在软件部署阶段，利用 TensorRT 或 ONNX Runtime 对模型进行量化（如FP16半精度推理）和加速，在保证精度的前提下显著降低端到端时延。 | 数据集准备与预处理阶段：配置Python、PyTorch及CUDA环境，选用 CIFAR-10 数据集作为核心训练集，以快速验证网络结构有效性；选用 Kodak 无损图像数据集作为最终测试集，用于评估系统在真实高分图像下的泛化性能。对训练数据进行归一化、随机裁剪和水平翻转处理，以增强模型的鲁棒性，防止过拟合。 / 模型设计与仿真：复现DeepJSCC基础模型，加入注意力模块改进编码器，嵌入AWGN/Rayleigh信道层进行联合训练。 / 基线系统实现：编写脚本模拟传统“JPEG+信道编码”的传输过程，记录该基线系统在不同信噪比下的误码率及图像解码成功率，作为对比基准。 / 软件系统集成：使用Streamlit或PyQt编写前端界面，后端调用模型推理接口，实现交互式功能。前端功能实现图像加载、信道参数（SNR值）滑块调节、传输模拟按钮；设计双屏显示区域，左侧显示原图，右侧实时显示经过DeepSC重构的图像和传统方案重构的图像。 / 测试与论文撰写：在AWGN和瑞利衰落信道下，分别测试本系统与基线系统的性能；收集实验数据，绘制PSNR/SNR性能曲线；分析语义特征传输相较比特流在被窃听环境下的解码难度与潜在隐私保护能力；分析语义通信减少重传、冗余编码带来的能耗节省；分析结果，撰写毕业论文。 / 3.3拟解决的主要问题及措施 / 在研究与实现过程中，预计会遇到以下关键技术难点，拟采取如下措施解决： / 物理信道的不可微性导致的端到端训练困难：实际无线信道（如衰落、硬件损伤）通常包含复杂的随机过程，直接将物理信道模型嵌入神经网络会导致梯度无法反向传播，无法进行端到端优化。 / 解决措施：采用 “生成式信道建模” 或 “重参数化技巧”。在训练阶段，不直接调用物理设备，而是使用数学解析模型来模拟信道传输过程，确保整个链路在数学上是可导的。同时，在推理/演示阶段，保持模型参数不变，验证其在模拟环境下的有效性。 / 深度语义模型在不同信噪比下的泛化能力差：通常针对特定SNR训练的模型，在信道条件改变时性能会大幅下降（即模型对SNR失配敏感）。若为每个SNR点单独训练模型，存储开销大且不实用。 / 解决措施：引入注意力机制辅助的信噪比感知网络。将当前的SNR值作为一个外部特征向量输入到编码器和解码器中，利用注意力模块动态调整网络权重。这使得单一模型能够感知信道状态，自动调整编码策略。 / 图像视觉质量评价与语义一致性的平衡：传统的MSE损失函数倾向于生成模糊平滑的图像，虽然信噪比数值尚可，但丢失了纹理细节（即视觉语义较差），导致“指标高但观感差”。解决措施：在损失函数中引入 SSIM（结构相似性）损失和感知损失（Perceptual Loss, 基于VGG网络提取特征），强制网络关注图像的高层语义特征而非仅仅是像素值差异。在GUI设计中，除了展示PSNR数值外，增加SSIM指标展示，并保留人工主观评估的窗口，确保系统优化的方向符合人类视觉感知。 / 复杂神经网络的计算开销与实时性矛盾：基于Transformer或深层CNN的模型计算量巨大，任务书要求在普通PC上实现低延迟推理，这对计算效率提出了挑战。 / 解决措施：模型轻量化：采用深度可分离卷积替代标准卷积，减少参数量。推理加速：在软件部署阶段，利用 TensorRT 或 ONNX Runtime 对模型进行量化（如FP16半精度推理）和加速，在保证精度的前提下显著降低端到端时延。 |
